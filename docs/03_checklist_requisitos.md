# Checklist de requisitos - Incremento Semana 11

Marca cada item despues de probarlo.

## A. Apertura e integracion
- [ ] La carpeta completa abre correctamente en Visual Studio Code.
- [ ] `index.html` carga sin errores visuales graves.
- [ ] `css/styles.css` esta enlazado correctamente.
- [ ] `js/app.js` esta enlazado correctamente.
- [ ] No aparecen errores de JavaScript en la consola despues de corregir.

## B. Formulario
- [ ] El formulario permite escribir nombre completo.
- [ ] El formulario permite escribir edad.
- [ ] El formulario permite escribir telefono.
- [ ] El formulario permite escribir correo.
- [ ] El formulario permite seleccionar taller.
- [ ] El formulario permite seleccionar jornada.
- [ ] La casilla de confirmacion funciona.

## C. Validaciones operativas
- [ ] Rechaza nombre vacio o menor a 3 caracteres.
- [ ] Rechaza edad menor a 12.
- [ ] Rechaza telefono con menos o mas de 10 digitos.
- [ ] Rechaza telefono con letras.
- [ ] Rechaza correo sin formato valido.
- [ ] Rechaza cuando no se selecciona taller.
- [ ] Rechaza cuando no se confirma la informacion.
- [ ] Acepta un registro que cumple todos los requisitos.

## D. Control de flujos
- [ ] Usa `if` / `else` para decidir si guarda o muestra errores.
- [ ] Usa `switch` para describir el taller seleccionado.
- [ ] Usa un ciclo `for` para recorrer registros o calcular datos.
- [ ] El flujo no guarda registros invalidos.
- [ ] El flujo muestra mensajes claros al usuario.

## E. Funcionamiento demostrable
- [ ] Los registros validos aparecen en la tabla.
- [ ] Se actualiza el total de inscritos.
- [ ] Se actualiza el total de registros validos.
- [ ] Se muestra el taller con mayor demanda.
- [ ] El boton limpiar funciona.
- [ ] El boton borrar registros funciona.
- [ ] Los registros persisten al recargar la pagina, si se mantiene localStorage.

## F. Calidad visual y usabilidad
- [ ] El sitio es legible.
- [ ] Los botones son claros.
- [ ] Los mensajes de error y exito se entienden.
- [ ] La tabla se puede leer.
- [ ] La pagina funciona en pantalla pequena o celular.

## G. Evidencias
- [ ] Se completo la matriz de pruebas.
- [ ] Se completo la bitacora de depuracion.
- [ ] Se completo el changelog del incremento.
- [ ] Se agregaron capturas del sitio funcionando.
